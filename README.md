# NailApp — Запись на маникюр
## 1. Тема и концепция

### Название
**NailApp**

### Краткое описание
NailApp — это приложение для записи на маникюр. Клиенты могут смотреть портфолио работ мастера, выбирать дизайн и записываться онлайн на свободное время. Мастер может добавлять фотографии своих работ и управлять записями клиентов. Всё в одном месте, без лишней переписки.

### Целевая аудитория
- Девушки и женщины, которые хотят записаться на маникюр.
- Мастера ногтевого сервиса, которым нужно удобно вести запись.

### Ключевой функционал
- Регистрация клиентов и мастера.
- Просмотр каталога работ с фотографиями.
- Онлайн-запись на выбранное время.
- Личный кабинет с историей записей.
- Сохранение понравившихся дизайнов в избранное.
- Админ-панель для мастера.

## 2. Структура приложения (8 разделов)

| № | Раздел | Путь | Тип | Описание |
|---|--------|------|-----|----------|
| 1 | **Главная** | `/` | Публичный | Приветствие, популярные дизайны, кнопка записи. |
| 2 | **Вход и регистрация** | `/auth/*` | Публичный | Вход, регистрация, восстановление пароля (считается за 1 раздел). |
| 3 | **Каталог дизайнов** | `/catalog` | Публичный | Фотографии работ мастера с фильтром по типу. |
| 4 | **Страница дизайна** | `/catalog/:id` | Публичный | Увеличенное фото, описание, цена, кнопка «Записаться». |
| 5 | **Запись** | `/booking` | Приватный | Выбор даты и времени, подтверждение записи. |
| 6 | **Личный кабинет** | `/profile` | Приватный | Предстоящие записи, история, избранное. |
| 7 | **Избранное** | `/favorites` | Приватный | Сохранённые дизайны. |
| 8 | **Админ-панель** | `/admin` | Приватный | Управление записями и галереей (только для мастера). |

## 3. Технологический стек

### Frontend
- **React** — популярная библиотека, позволяет сделать сайт быстрым и удобным.
- **React Router** — для переходов между страницами без перезагрузки.
- **CSS** — для оформления внешнего вида.

### Backend
- **Node.js + Express** — чтобы сервер работал на JavaScript, удобно учить один язык.
- **SQLite** — простая база данных, хранится в одном файле, не требует установки.

### Инфраструктура
- **Frontend:** GitHub Pages или Vercel (бесплатно).
- **Backend:** Render (бесплатный хостинг).
- **База данных:** файл SQLite в проекте.

## 4. Описание страниц

### Страница 1: Главная (`/`)
**Тип:** Публичный

**Блоки:**
1. Шапка: логотип «NailApp», меню (Главная, Каталог, Войти).
2. Большой блок с фотографией маникюра и кнопкой «Записаться сейчас».
3. Секция «Популярные дизайны»: 6 карточек с фото (данные загружаются с сервера).
4. Футер: контакты мастера, ссылка на соцсети.

**Состояния:**
- *Загрузка:* серые прямоугольники вместо фото.
- *Данные:* фотографии с подписями.
- *Ошибка:* «Не удалось загрузить работы. Попробуйте позже.»

### Страница 2: Вход и регистрация (`/auth/*`)
**Тип:** Публичный

**Блоки:**
1. Упрощённая шапка с логотипом.
2. Форма входа: поля «Email», «Пароль», кнопка «Войти».
3. Ссылка «Забыли пароль?».
4. Ссылка «Нет аккаунта? Зарегистрироваться».

**Состояния:**
- *Ошибка:* «Неверный email или пароль», красная обводка полей.
- *Успех:* переход на главную страницу.

### Страница 3: Каталог дизайнов (`/catalog`)
**Тип:** Публичный

**Блоки:**
1. Шапка стандартная.
2. Кнопки-фильтры: «Все», «Гель-лак», «Наращивание», «Дизайн».
3. Сетка из карточек с фотографиями и названиями.
4. Пагинация внизу страницы.

**Состояния:**
- *Загрузка:* 12 серых скелетонов карточек.
- *Данные:* сетка с фото.
- *Пусто:* «В этой категории пока нет работ.»
- *Ошибка:* «Ошибка загрузки. Попробуйте обновить страницу.»

### Страница 4: Страница дизайна (`/catalog/:id`)
**Тип:** Публичный

**Блоки:**
1. Шапка стандартная.
2. Крупное фото дизайна.
3. Название, описание, цена, длительность.
4. Кнопки «Записаться с этим дизайном» и «В избранное».
5. Секция «Похожие дизайны».

**Состояния:**
- *Загрузка:* спиннер по центру.
- *Данные:* фото и информация.
- *Ошибка:* «Дизайн не найден.»

### Страница 5: Запись (`/booking`)
**Тип:** Приватный (только для клиентов)

**Блоки:**
1. Шапка стандартная.
2. Выбор услуги из списка.
3. Календарь с доступными датами.
4. Кнопки с временем: свободные слоты активны, занятые — серые.
5. Кнопка «Подтвердить запись».

**Состояния:**
- *Нет слотов:* «На эту дату всё занято. Выберите другую.»
- *Успех:* «Вы записаны!» и переход в личный кабинет.

### Страница 6: Личный кабинет (`/profile`)
**Тип:** Приватный

**Блоки:**
1. Шапка стандартная.
2. Имя и телефон пользователя, кнопка «Редактировать».
3. Вкладка «Предстоящие записи»: дата, время, дизайн, кнопка «Отменить».
4. Вкладка «История»: список прошлых записей.

**Состояния:**
- *Нет записей:* «У вас пока нет записей. Запишитесь прямо сейчас!»

### Страница 7: Избранное (`/favorites`)
**Тип:** Приватный

**Блоки:**
1. Шапка стандартная.
2. Сетка сохранённых дизайнов с кнопкой «Удалить из избранного».

**Состояния:**
- *Пусто:* «В избранном пока ничего нет. Перейдите в каталог.»
- *Ошибка:* «Не удалось загрузить избранное.»

### Страница 8: Админ-панель (`/admin`)
**Тип:** Приватный (только для мастера)

**Блоки:**
1. Шапка с пометкой «Режим мастера».
2. Вкладка «Записи»: таблица с датой, временем, клиентом, статусом. Кнопки «Подтвердить» и «Отменить».
3. Вкладка «Галерея»: кнопка «Добавить дизайн», форма для загрузки фото, список текущих работ с кнопкой «Удалить».

**Состояния:**
- *Нет записей:* «На сегодня записей нет.»
- *Успех:* уведомление «Запись подтверждена!»

## 5. Схема данных и API

### Модели данных

**User (Пользователь)**
- id — число
- name — строка
- email — строка
- password — строка
- phone — строка
- role — строка (client или master)

**Design (Дизайн)**
- id — число
- title — строка
- description — текст
- category — строка
- price — число
- duration — число (минут)
- imageUrl — строка

**Booking (Запись)**
- id — число
- userId — число
- designId — число
- date — строка
- time — строка
- status — строка (pending, confirmed, cancelled)

**Favorite (Избранное)**
- id — число
- userId — число
- designId — число

### Эндпоинты API

| Метод | Путь | Описание | Доступ |
|-------|------|----------|--------|
| `POST` | `/api/auth/register` | Регистрация | Публичный |
| `POST` | `/api/auth/login` | Вход | Публичный |
| `GET` | `/api/designs` | Список дизайнов | Публичный |
| `GET` | `/api/designs/:id` | Один дизайн | Публичный |
| `GET` | `/api/slots` | Свободные слоты | Приватный |
| `POST` | `/api/bookings` | Создать запись | Приватный |
| `GET` | `/api/bookings/my` | Мои записи | Приватный |
| `GET` | `/api/admin/bookings` | Все записи | Приватный (мастер) |
| `PATCH` | `/api/admin/bookings/:id` | Обновить статус | Приватный (мастер) |
| `POST` | `/api/favorites/:designId` | В избранное | Приватный |
| `DELETE` | `/api/favorites/:designId` | Удалить из избранного | Приватный |
